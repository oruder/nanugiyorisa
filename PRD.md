# 나누기 학습 프로그램 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
**나누기 요리사** (Division Chef)

### 1.2 목표
초등학교 4학년 학생들이 분수와 나눗셈 개념을 요리 시뮬레이션을 통해 재미있게 학습할 수 있는 교육용 프로그램

### 1.3 타겟 사용자
- 초등학교 4학년 학생 (만 9-10세)
- 분수와 나눗셈 개념을 학습 중인 학생

---

## 2. 핵심 컨셉

### 2.1 학습 메커니즘
요리해서 서빙하는 과정을 통해 나눗셈과 분수 개념을 자연스럽게 학습

### 2.2 게임 플로우
```
주문 확인 → 재료 선택 → 재료 등분 → 접시에 담기 → 서빙 → 결과 확인
```

## 3. 기능 요구사항

### 3.1 주문 화면 (상단 영역)

#### 3.1.1 주문 정보 표시
- **위치**: 화면 상단
- **표시 내용**:
  - 요구되는 재료 종류 (예: 생선, 고기, 채소)
  - 요구되는 재료 수량
    - 자연수 형태 (예: 생선 2마리)
    - 분수 형태 (예: 생선 2/4 마리)
- **분수 표기법**: 세로 형태로 표기 (공책 표기 방식)
  ```
  2
  ―
  4
  ```

#### 3.1.2 주문서 디자인
- 주문서 형태의 UI
- 명확하고 읽기 쉬운 글꼴
- 재료별 아이콘 표시

### 3.2 재료 창고 영역

#### 3.2.1 재료통 종류
- 생선통
- 고기통
- 채소통
- (추가 재료 확장 가능)

#### 3.2.2 재료 가져오기 기능
- **인터랙션**: 재료통 클릭 또는 드래그
- **기능**: 원하는 재료를 원하는 개수만큼 가져오기
- **제한**: 없음 (학생이 자유롭게 선택)
- **피드백**: 가져온 재료가 작업대로 이동

### 3.3 작업대 영역

#### 3.3.1 재료 등분 기능
- **트리거**: 가져온 재료 클릭
- **입력 방식**: 숫자 입력창 표시 (몇 등분할지 입력)
- **입력 범위**: 2~12 등분 (4학년 수준 고려)
- **실행**: 확인 버튼 클릭 시 재료가 등분됨

#### 3.3.2 재료 분할 비주얼
- **재료 형태**: 
  - 재료는 실제 모양을 유지 (네모 박스 금지)
  - SVG 또는 이미지를 활용한 사실적인 재료 표현
  - 생선, 고기, 채소 등 각 재료의 고유한 형태 유지
- **애니메이션**: 재료가 쪼개지는 애니메이션 효과
- **시각적 표현**: 
  - 칼로 자르는 효과 (실제로 자르는 듯한 애니메이션)
  - 명확하게 구분되는 절단선
  - 각 조각은 원래 재료의 축소/분할된 형태 유지
- **분할 결과**: 
  - 동일한 크기의 조각들로 나뉨
  - **중요**: 큰 생선이 작은 생선 여러 마리가 되는 것이 아니라, 반 마리씩 나뉨
  - 오개념 방지: 전체를 나눈 부분임을 명확히 시각화
- **오개념 방지 원칙**:
  - 1마리를 2등분 → 0.5마리 크기의 조각 2개 (작은 생선 2마리 ✗)
  - 1개를 4등분 → 0.25개 크기의 조각 4개 (작은 것 4개 ✗)
  - 절단선으로 명확히 분할되었음을 표시

#### 3.3.3 조각 선택 기능
- **인터랙션**: 분할된 조각 클릭 또는 드래그
- **선택 표시**: 선택된 조각 하이라이트
- **다중 선택**: 여러 조각 선택 가능
- **취소**: 선택 해제 가능

### 3.4 접시 영역

#### 3.4.1 접시에 담기
- **기능**: 선택한 조각들을 접시로 이동
- **방법**: 
  - 드래그 앤 드롭
  - "접시에 담기" 버튼
- **시각적 표현**: 접시에 담긴 재료가 정렬되어 표시

#### 3.4.2 담긴 재료 관리
- **수량 표시**: 현재 접시에 담긴 양을 분수로 표시 (세로 분수 형태)
  - 가로 표기 (1/5) 금지
  - 세로 표기만 사용:
  ```
  1
  ―
  5
  ```
- **수정 가능**: 접시에서 재료 제거 가능
- **실시간 계산**: 담긴 양 자동 계산 및 표시

### 3.5 서빙 및 검증

#### 3.5.1 서빙 기능
- **트리거**: "서빙하기" 또는 "보내기" 버튼
- **조건**: 접시에 재료가 담겨있을 때 활성화

#### 3.5.2 정답 검증
- **비교 로직**: 
  - 주문한 양 = 서빙한 양 → 성공
  - 주문한 양 ≠ 서빙한 양 → 실패
- **분수 비교**: 
  - 2/4 = 1/2 (약분 고려)
  - 4/4 = 1 (가분수와 자연수 변환)

#### 3.5.3 피드백
- **성공 시**:
  - 성공 애니메이션 (별, 하트 등)
  - 칭찬 메시지
  - 점수 또는 스타 획득
  - 다음 문제로 이동 버튼
  
- **실패 시**:
  - 부드러운 실패 메시지 (격려 포함)
  - 힌트 제공 옵션
  - "다시 시도하기" 버튼
  - 현재 상태 유지 또는 초기화 선택

## 4. 학습 설계

### 4.1 난이도 단계

#### Level 1: 기초
- 자연수 나눗셈 (예: 생선 2마리)
- 2등분, 4등분만 사용
- 단순 분수 (1/2, 1/4)

#### Level 2: 중급
- 단위 분수 (1/3, 1/4, 1/6)
- 3등분, 6등분 추가
- 여러 재료 조합 (1-2가지)

#### Level 3: 고급
- 진분수 (2/3, 3/4, 5/6)
- 8등분, 12등분 추가
- 여러 재료 조합 (2-3가지)
- 약분 필요한 문제

#### Level 4: 심화
- 가분수와 대분수 (5/4 = 1 1/4)
- 복잡한 등분 조합
- 3가지 이상 재료

### 4.2 학습 목표
- **나눗셈 이해**: 전체를 동등하게 나누는 개념
- **분수 이해**: 부분과 전체의 관계
- **분수 연산**: 분수의 덧셈과 동치
- **약분 개념**: 다른 표현의 같은 양
- **문제 해결**: 목표 달성을 위한 전략 수립

## 5. UI/UX 요구사항

### 5.0 분수 표기 원칙
**중요**: 프로그램의 모든 분수는 세로 형태로만 표기
- 가로 표기 (1/5, 2/3 등) 사용 금지
- 세로 분수 형태만 사용:
  ```
  분자
  ――
  분모
  ```
- 적용 범위: 주문서, 조각 라벨, 접시 수량 표시, 힌트 등 모든 UI

### 5.1 화면 구성
```
┌─────────────────────────────────────┐
│     주문서 (요구 재료 및 수량)      │
├─────────────────────────────────────┤
│                                     │
│  [재료통들]         [작업대]        │
│                                     │
│  🐟 생선통         [선택한 재료]    │
│  🥩 고기통         [등분 버튼]      │
│  🥬 채소통                          │
│                     [접시]          │
│                    [서빙하기]       │
│                                     │
└─────────────────────────────────────┘
```

### 5.2 디자인 원칙
- **직관적**: 4학년 학생이 설명 없이 사용 가능
- **친근함**: 밝고 귀여운 색상과 일러스트
- **명확성**: 큰 버튼, 명확한 라벨, 읽기 쉬운 글꼴
- **피드백**: 모든 행동에 즉각적인 시각/청각 피드백

### 5.3 인터랙션
- **드래그 앤 드롭**: 재료 이동의 주요 방법
- **클릭**: 선택 및 실행
- **숫자 입력**: 등분 개수 입력 (터치 키보드 지원)
- **애니메이션**: 부드럽고 명확한 전환 효과

### 5.4 접근성
- **색상 대비**: WCAG 기준 충족
- **텍스트 크기**: 최소 14pt 이상
- **음성 지원**: 선택적 텍스트 읽기 기능
- **키보드 지원**: 마우스 없이도 조작 가능

## 6. 기술 요구사항

### 6.1 플랫폼
- **우선순위**: 웹 브라우저 (HTML5)
- **호환성**: 
  - Chrome, Edge, Firefox, Safari 최신 버전
  - 태블릿 및 데스크톱 지원
- **반응형**: 다양한 화면 크기 대응

### 6.2 기술 스택 (권장)
- **프론트엔드**: React, Vue, 또는 Vanilla JS
- **그래픽**: Canvas 또는 SVG
- **애니메이션**: CSS Animation, GSAP, 또는 Anime.js
- **상태 관리**: Redux, Vuex, 또는 Context API

### 6.3 성능 요구사항
- 초기 로딩: 3초 이내
- 인터랙션 반응: 100ms 이내
- 애니메이션: 60fps 유지

## 10. 성공 지표

### 10.1 학습 효과
- 문제 정답률 70% 이상
- 레벨 완료율 80% 이상
- 평균 학습 시간 20-30분/세션
